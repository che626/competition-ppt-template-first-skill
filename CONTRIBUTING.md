# Contributing

Contributions should make the Skill more reliable for real competition decks, not merely add decoration.

## Good contributions

- A reusable, domain-aware layout archetype with a clear evidence use case.
- A prompt pattern that creates usable editable zones without generic empty frames.
- A rendered QA check that catches a real production failure.
- A sanitized, rights-cleared example showing input, template plan, editable layers, and output.
- Improvements to installation, task state, or approval gates.

## Do not submit

- Private reports, certificates, system screenshots, competition submissions, API keys, or user data.
- Copyrighted PPT templates or visual assets without redistribution rights.
- Generic card collections with no narrative or evidence purpose.
- Claims about benchmarks, awards, or deployment that are not verifiable.

## Pull-request checklist

- [ ] The addition has a clear use case and does not duplicate an existing reference.
- [ ] Examples are generic and contain no private evidence.
- [ ] Instructions work for a Chinese competition deck when Chinese is the target language.
- [ ] The change preserves the template-first principle and editable evidence layers.
- [ ] Markdown links resolve locally.
