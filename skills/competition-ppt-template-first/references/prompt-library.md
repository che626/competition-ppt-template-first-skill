# Template-Image Prompt Library

These prompts are starting structures, not fixed styles. Replace bracketed fields with facts from the project blueprint. Always keep final text, metrics, charts, screenshots, and certificates out of the generated image; those are added as editable PPT layers.

## Shared prompt tail

Append this to every template-image prompt:

```text
16:9 presentation background template, high visual completion, deliberate low-detail zones for editable Chinese text and real evidence images, integrated composition rather than a collage of empty cards. No readable text, no fake dashboard screenshot, no watermark, no generic HUD wireframe, no repeated rounded-card wall, no oversized decorative sphere.
```

## Cover

```text
Premium competition-defense cover for [project domain]. Cinematic [real subject scene], with [project-specific device/material/process] as the first visual signal. Strong asymmetric focal point, ample breathing room for a large Chinese project title, restrained metadata zone, [palette and material]. The image should feel confident and technically grounded, not cute or marketing-like.
```

## Industry pain point

```text
Premium competition-defense page explaining why [industry workflow] fails today. Show an authentic-feeling [work scene / process / material] with three visually integrated zones for: inconsistency, broken records, and difficult review. Use a central visual tension and a calm conclusion strip. [palette and material].
```

## Design scheme / transformation

```text
Premium competition-defense page showing [input] transformed by [AI vision / engineering process] into [structured result / action]. Center a visually credible input or result image; reserve a left evidence zone, a right outcome zone, and a bottom closed-loop conclusion. Use [domain scene, material, and lighting] to connect the process instead of drawing a generic flowchart.
```

## Model choice / comparison

```text
Premium technical competition-defense page for a verified model-selection decision in [domain]. Reserve a quiet, high-contrast decision matrix zone for editable comparison facts, a real-result evidence zone, and a small deployment-chain zone. Background should be dark and restrained so dense technical copy remains readable; use [domain-specific visual motif] rather than a generic data screen.
```

## Iteration

```text
Premium competition-defense page showing three stages of [model or system] iteration. Use a continuous visual evolution through [real artifacts or project-specific material], with three unequal but aligned evidence zones and a clear bottom statement about what the iteration unlocked. Avoid three identical boxes connected by a simple arrow.
```

## System screenshots

```text
Premium competition-defense page for real [product/system] screenshots. The supplied screenshots must become the main visual anchors, with a calm dark template, integrated device/material context, concise label zones, and clear hierarchy. Do not generate a fake interface. Reserve large clean frames matching the screenshots' aspect ratios.
```

## Evidence wall / awards

```text
Premium competition-defense evidence page for real papers, certificates, software records, or test artifacts. Create an asymmetrical evidence-wall composition with large readable artifact regions and a right-side argument column explaining credibility. Use restrained high-end lighting and material; reserve enough space for real documents without forcing them into tiny decorative windows.
```

## Future roadmap

```text
Premium competition-defense roadmap page for [project domain]. Present completed work, in-progress work, and future scenario expansion as a grounded progression through real domain imagery. Reserve five concise editable milestone zones with varied visual weight; do not use a generic five-card timeline.
```
