# Layout Registry

Complete this before generating templates. The registry keeps a deck visually varied without losing its shared token system.

```md
| Slide | Page role | Page rhythm | Primary claim | Layout family | Reading order | Entry point / visual anchor | Real-asset slot | Text budget | Contrast mode | Structural relation to previous page | Break pattern needed? | Status |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 01 | cover | anchor |  | hero identity | title -> promise -> subject | project / product hero |  | low | dark / rich | opening | no | planned |
| 02 | problem | dense |  | evidence-led asymmetry | claim -> evidence -> implication | real scene / sample | E-01 | medium | mid | changes from hero | no | planned |
| 03 | method | breathing |  |  |  |  |  |  |  |  | no | planned |
```

## Registry rules

- A layout family is the dominant reading structure, not the background color. Examples: hero identity, evidence-led asymmetry, central transformation, comparison matrix, loop, screenshot cluster, editorial split, evidence wall, roadmap progression.
- Set one page rhythm for every row: `anchor` for a strong visual/decision focal point, `dense` for multi-evidence proof that still scans in blocks, or `breathing` for a low-load reset, transition, or image-led page. Rhythm changes density and visual weight; it does not exempt a page from its template contract.
- Do not repeat the same layout family more than twice in a row. Use a structural break when the narrative changes from problem to response, proof to product, or product to future.
- Record the actual reading order and visual entry point. A page may be dense, but it must still guide the eye deliberately.
- A layout change must preserve the deck tokens: title origin, type scale, spacing, image treatment, and accent logic remain stable unless a recorded exception is justified.
- Do not choose a layout because it is convenient to fill with cards. Choose it because it gives the page's claim, evidence, and word budget the correct visual weight.
