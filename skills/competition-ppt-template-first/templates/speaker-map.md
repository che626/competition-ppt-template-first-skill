# Speaker Map

Complete this after the slide-source map and before final slide copy. It keeps the slide useful to the audience instead of turning it into the presenter’s script.

```md
| Slide | Judge takeaway | Audience sees on the PPT | Presenter adds aloud | Time budget | Transition / likely question | Backup material | Status |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 01 |  | title, promise, visual anchor | project context and opening hook |  | why the project matters |  | planned |
| 02 |  | sourced problem and evidence | operating context and implication |  | how the response follows |  | planned |
```

## Rules

- Put the judgment, authentic evidence, and essential labels on the slide. Put explanation, interpretation, transitions, and nonessential detail in the presenter column.
- Do not create a long script on the visible slide. If audience reading would compete with speaking, shorten the visible copy or move detail to a backup page.
- Every spoken metric, model comparison, award, or capability claim must still be grounded in the fact registry or the data-evidence plan.
- Record likely judge questions and the backup artifact that answers them. Do not fabricate an answer to fill this column.
- Use the time budget to keep the complete defense inside the required duration.
