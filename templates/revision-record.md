# Revision Record

Create one entry for every material visual revision.

```md
## [YYYY-MM-DD] Slide [NN] - [title]

- State before revision: planned / template-review / build-review / approved / retired
- User feedback:
- Root cause: content / composition / underlay template / evidence / typography / factual claim
- Structural decision:
- Files retired:
- Files created:
- QA result:
- State after revision:
```

## Rules

- "Move an object" is not a root cause. State why the original composition failed.
- If the underlay template is the root cause, replace it and record the old version as retired.
- If a page is approved, preserve its exact path and do not alter it without a new revision record.
